# Yunjae Review ver1 — 8관점 문서 리뷰

내가 만든 보고서·자료·공지문을 **8개 이해관계자 관점**에서 검토하고, 관점별 피드백 + 우선 수정안을 낸다.
좋게만 말하지 않는다. 각 관점이 **진짜 신경 쓸 지점**을 원문 문장·문단을 지목해 구체적으로 짚는다.

## 진행 절차
1. **문서 확보** —
   - **붙여넣기·첨부**: 그대로 사용.
   - **구글 드라이브 링크**: 링크에서 fileId 추출 → `mcp__claude_ai_Google_Drive__read_file_content`. 미인증이면 `/mcp` 재인증 안내.
   - **웹 URL**: `WebFetch`로 본문 확보.
   - **문서가 없으면**: 무엇을 리뷰할지 먼저 묻는다(추측 금지).
2. **맥락 파악** — 문서 종류(보고서/자료/공지문)·목적·독자·민감도를 1줄로 정리. 불명확하면 1~2개만 컨펌.
3. **8관점 리뷰** — 아래 관점표 기준으로 각각 검토.
4. **통합** — 공통 지적·관점 충돌을 모아 우선순위 수정안 작성.

## 8관점 (누적 구조 — 아래로 갈수록 위 관점을 포함하며 고유 목표가 추가됨)
| # | 관점 | 판단 기준(이 관점이 통과시키려는 것) |
|---|---|---|
| 1 | HR팀 구성원 | **오타·표기·형식** + HR 실무 정확성(빠진 디테일, 바로 실행 가능한지) |
| 2 | HR 직책자(팀장) | 1번 전부 + **전사적으로 괜찮은가** + **임원을 설득할 수 있는가** |
| 3 | HR 임원 | 2번 전부 + **대표이사를 설득해 승인받을 수 있는가** |
| 4 | 비HR 구성원 | 이 제도·규정·공지가 **납득할 만한가** (이해 용이성·형평성·수용성) |
| 5 | 비HR 직책자 | 4번 전부 + **내 조직 운영에 괜찮은가** (부담·형평·협조요청 명확성) |
| 6 | 비HR 임원 | 5번 전부 + **내 조직·전사 관점에서 괜찮은가** (우선순위·비용효익) |
| 7 | 대표이사 | 이 제도가 **회사에 도움이 되는가** (전략·평판·법적 리스크·ROI, 의사결정 포인트) |
| 8 | 외부 제3자 | 객관성, 대외 평판·법적 노출, 오해 소지, 컴플라이언스·개인정보 |

각 관점마다: **한줄 인상 → 강점 → 우려/리스크 → 구체 수정 제안 → 수용도(높음/보통/낮음)**. 막연한 칭찬 금지, 어느 문장·문단을 어떻게 고칠지 지목한다. 누적 관점(2·3·5·6)은 아래 단계의 핵심을 다시 늘어놓지 말고 **새로 추가되는 판단(설득 가능성·조직 영향 등)에 집중**한다.

```
## 📌 총평

## 👥 관점별 피드백

- 수정 제안: …(원문 어느 부분을 어떻게)
### 2. HR 직책자 (팀장급)  · 수용도: ○○
…
(8. 외부 제3자까지 동일 구조)

## ⚠️ 관점 충돌
상반된 요구(예: 임원은 간결, 실무자는 상세)를 짚고 절충안 제시.

## ✅ 우선 수정 체크리스트
영향 큰 순서로. 각 항목에 [필수]/[권장]/[선택] 태그 + 관련 관점 번호.
```

## 마무리
- **민감정보 경고**: 개인정보·미공개 수치·법적 표현이 대외 노출 위험이면 8번 관점에서 강하게 경고.
- 문서 목적·독자가 불명확하면 추측 말고 먼저 확인.
- 길면 관점별 핵심만. "○번 관점만 깊게" 요청 시 해당 관점 집중.
- 칭찬·비판 균형보다 **개선에 도움 되는 솔직함** 우선.
